# 主题优化任务计划

## 目标

在不破坏现有水墨视觉风格的前提下，补齐 `valaxy-theme-shuimo` 的配置兑现度、渲染一致性、性能稳定性与工程可验证性，并为后续功能扩展建立清晰路线。

| 阶段 | 名称 | 状态 | 说明 |
|------|------|------|------|
| P0-1 | 配置兑现度审计 | completed | 已完成第一轮配置兑现与支持矩阵梳理 |
| P0-2 | 文章页渲染收敛 | completed | 已统一到 `theme/layouts/post.vue`，删除未引用组件 |
| P0-3 | 最小验证闭环 | completed | 已接入 `pnpm test`（vitest）并补齐 `theme/node` 的单元测试 |
| P1-1 | 首页性能治理 | completed | 山水仅首页挂载、生成走调度器、会话内按尺寸缓存 |

| P2-2 | 固定种子与可复现首页 | pending | 支持锁定山水 seed 与刷新策略 |

| P2-4 | Demo 实时配置面板 | pending | 提升展示与人工回归效率 |

## 分阶段计划

### P0-1 配置兑现度审计

- 建立配置清单：`theme/types/index.d.ts`、`theme/node/index.ts`、`README.md`
- 标记每个配置项为“已实现 / 部分实现 / 未实现”
- 优先核对：`colors.*`、`fonts.*`、`footer.since`、`decorations.opacity`、`stamp.shape`
- 处理策略：
  - 已声明但未实现：补接运行时
  - 暂不支持：从类型或文档中移除，避免假承诺

验收标准：
- README、类型声明、默认配置、运行时行为一致
- 新增一个“配置支持矩阵”供后续维护

### P0-2 文章页渲染收敛

- 梳理 `theme/layouts/post.vue` 与 `theme/components/ShuimoArticle.vue` 的职责重叠
- 选定唯一文章渲染主入口
- 将标题、元信息、正文、印章、上下篇导航统一到同一条渲染链路
- 抽离可复用片段，避免页面级样式与组件级样式重复定义

验收标准：
- 文章页只保留一条主实现路径
- 修改文章样式时不需要同时改两个文件

### P0-3 最小验证闭环

- 明确发布前最低检查：`pnpm lint`、`pnpm typecheck`、`pnpm build`
- 增加 `test` 脚本入口，优先使用 Vitest
- 补 2-3 个高价值测试：
  - 配置合并/默认值
  - 关键组件渲染
  - demo 构建与关键页面 smoke check

验收标准：
- 本地与 CI 可以一键执行最小回归
- 主题发布不只依赖人工预览

### P1-1 首页性能治理

- 仅在需要页面挂载 `ShuimoHeroLandscape`
- 将重计算接入调度器，避免主线程连续阻塞
- 为山水生成与宣纸纹理增加更稳定的缓存策略
- 评估是否需要首屏预生成或懒生成

验收标准：
- 路由切换与首页首屏不卡顿
- 低性能设备可接受

### P1-2 动效降级与可访问性

- 为幕布动画、主题切换、装饰动效增加 `prefers-reduced-motion`
- 限制幕布动画触发场景，避免每次路由切换都重播
- 为导航、按钮、印章链接补键盘焦点样式
- 检查深色模式下文字与背景对比度

验收标准：
- 动效可关闭、焦点可见、暗色可读

### P2 功能扩展

- `P2-1` 主题预设：`classic / night / gold / minimal`
- `P2-2` 固定 seed：首页山水可复现、可锁定
- `P2-3` 阅读增强：目录、阅读时长、图片题注、脚注美化
- `P2-4` Demo 面板：实时切换字体、颜色、印章、宣纸和山水参数

验收标准：
- 新功能优先服务真实博客场景
- 配置和 demo 同步更新

## 风险与依赖

| 风险 | 影响 | 应对 |
|------|------|------|
| 类型声明与实际实现长期不一致 | 用户配置无效，维护成本上升 | 先做配置矩阵审计 |
| 文章渲染存在双实现 | 样式和功能改动易漏改 | 尽早收敛唯一入口 |
| 首页生成逻辑过重 | 首屏或切页卡顿 | 调度、缓存、按需挂载 |
| 缺少自动化验证 | 发布回归依赖人工 | 建立最小测试闭环 |

## 建议执行顺序

1. P0-1 配置兑现度审计
2. P0-2 文章页渲染收敛
3. P0-3 最小验证闭环
4. P1-1 首页性能治理
5. P1-2 动效降级与可访问性
6. P2 系列按产品价值逐步推进
